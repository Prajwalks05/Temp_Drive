Secure File Sharing System

System Architecture

This application is built using a 100% serverless, event-driven architecture on AWS. It uses Pre-Signed URLs to bypass the API compute layer for heavy file transfers, and EventBridge Scheduler for zero-compute auto-expiry.

How Data is Stored

Amazon S3 (The Files)

Physical files are stored in a private S3 bucket. The bucket blocks all public access. Files can only be written or read using temporary, cryptographically signed URLs generated by the Lambda functions.

Amazon DynamoDB (The Metadata)

Stores the "blueprint" of the file. Each record contains:

  • file_id (Partition Key)
  • owner_id (Cognito User SUB)
  • shared_with (Array of user IDs)
  • expires_at (Timestamp)

API Integration Guide

Authentication Requirement: All API Gateway endpoints (except the direct S3 PUT) require a valid Cognito IdToken passed in the Authorization header as a Bearer Token.
POST {base_url}upload

Authorizes a file upload, creates a database record, sets the auto-expiry trigger, and returns a pre-signed S3 URL.

Headers:
  • Authorization: Bearer <IdToken>
Request Body (JSON): 
{
  "filename": "confidential_report.pdf",
  "contentType": "application/pdf",
  "expiryHours": 24
}
Response (200 OK): 
{
  "file_id": "uuid-1234-5678",
  "upload_url": "{upload_url}",
  "message": "Upload link generated. Expires in 15 minutes."
}
PUT {upload_url}

Uploads the physical file directly to the S3 bucket bypassing the backend server. The {upload_url} is the massive string returned from the previous POST request.

CRITICAL: Do NOT send the Authorization header for this request. S3 will reject it.
Headers:
  • Content-Type: (Must match what you sent in Step 1)
  • No Auth Headers
Request Body:

Raw Binary File Data

Response:

200 OK (Empty body)

POST {base_url}share

Updates DynamoDB to grant download permissions to another Cognito user.

Headers:
  • Authorization: Bearer <IdToken>
Request Body (JSON): 
{
  "file_id": "uuid-1234-5678",
  "target_user_id": "cognito-sub-id-of-recipient"
}
Response (200 OK): 
{
  "message": "File shared successfully."
}
GET {base_url}download/{file_id}

Verifies if the requester is the owner or in the shared_with list. If authorized, returns a temporary pre-signed S3 download link.

Headers:
  • Authorization: Bearer <IdToken>
Response (200 OK): 
{
  "download_url": "https://s3.amazonaws.com/...",
  "message": "Link valid for 15 minutes."
}
